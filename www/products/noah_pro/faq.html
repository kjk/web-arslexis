Q1. What's in a name?

Noah Webster was a famous lexicographer.

Q2. Program fails to install.

Some databases are quite big. Please check that you have enough free memory on your PalmPilot. If there is not enough memory synchronization will fail.

Q3. I use Macintosh and I got garbage on the screen or program crashes.

A few Mac users reported that what they got on the screen is garbage or the program crashes. This is most likely fault of the unpacker program used to unzip noah_pro.zip.

Some unzippers have settings to convert text files to Mac text files. This alters the content of files and corrupts them. This option can be set to "convert if file is recognized as text" but unfortunately .prc and .pdb files are often (wrongly) recognized as text. To fix that change your unzipper program settings to "never convert text files".

For ZipIt 1.4 the preference reads "by default, smart-strip linefeeds from text files" and should be turned off. It may be different in different programs.

Q4. I get "Database not found" error on startup

It means that only the reader program has been installed (noah_pro.prc). Please also install a database (e.g., wn_medium.pdb) on your device.

Q5. Does Noah support Springboard memory extension for Visor?

Yes, you can put Noah Pro on Visor's 8 MB flash module (but not the backup module). Just move Noah Pro and a databases of your choice to flash module using File Mover application.

Important: it is very important not to move Noah Pref to flash module. If you do it Visor will crash with "Fatal error.......dataMgr.c,Line:5304,DmWriteCheck" error when you start Noah. To fix it just move Noah Pref back to regular memory.

Q6. Does Noah support TGRpro's compact flash?

Noah runs on TGRpro just fine but it doesn't know how to use it's compact flash memory. It also may crash if it's placed on CF card and used with AutoCF

Ability to read databases directly from flash memory is one of the planned future improvements for Noah.

Q7. The database is too big.

We want to provide the biggest and most comprehensive dictionary for PalmPilot. Even though Noah Pro uses very efficient compression (more than 50%) the size of the source material makes the resulting database big. But we also provide a lot of smaller dictionaries. Read database comparison to decide which database will best suit your needs.
